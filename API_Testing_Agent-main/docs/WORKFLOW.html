ğŸ”„ Complete Workflow - Step by Step

Phase 1: Verification

Command:
python run_agent.py check
What It Does: Validates that all prerequisites are installed and configured correctly. Input: None (reads your system state) Process:
  • Checks Ollama connection
  • Verifies Llama3 model availability
  • Validates spec files exist
  • Confirms pytest is installed
Output: Green âœ“ or Red âœ— for each check

Phase 2: Test Generation & Mock Data

Command:
python run_agent.py generate --resource agents --mock-agents 5
What It Does: Generates comprehensive test cases using Llama3 and creates realistic mock data. Input:
  • --resource: agents | policies | commissions | payments | performance
  • --mock-agents: Number of agents to generate (default: 5)
  • --output: Output directory (optional)
Process:
  • Reads OpenAPI specifications
  • Sends to Llama3 AI with prompt: "Generate comprehensive test cases"
  • Receives generated Python test code
  • Generates mock data (realistic names, emails, premiums)
  • Saves test files to generated_tests/tests/python/
Output:
  • test_agents.py (50-80+ test cases)
  • mock_agents.json (realistic test data)

Phase 3: Test Execution

Command:
python run_agent.py run --pattern test_agents
What It Does: Executes pytest tests and collects results. Input:
  • --pattern: Filter tests by name (optional)
Process:
  • Discovers test files in generated_tests/
  • Executes pytest with coverage
  • Parses results (PASSED, FAILED, SKIPPED)
  • Calculates success rate and execution time
Output:

Success Rate: 95.7%
Detailed failure information

Phase 4: Report Generation

Command:
python run_agent.py full --resource agents
What It Does: Runs complete workflow: generate â†’ run â†’ report. Input: Resource type and mock data count Process:
  • Executes all previous phases in sequence
  • Collects all results
  • Runs validators (schema, business logic, security)
  • Generates professional reports
Output:
  • test_report.html (interactive with charts)
  • test_report.json (machine-readable)
  • test_report.md (GitHub-friendly)
